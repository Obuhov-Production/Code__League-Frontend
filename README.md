# Code League Frontend


Frontend-частина платформи **Code League** для проведення IT-турнірів.


Проєкт реалізує публічний сайт платформи та особистий кабінет користувача з ролями, командами, турнірними, рейтингами, чатом і адміністративними інструментами.


## Про проєкт
**Code League** — це вебплатформа для організації та супроводу турнірів, де учасники можуть:


- реєструватися та входити в систему;
- підтверджувати email і використовувати OAuth-авторизацію;
- переглядати турніри та лідерборди;
- створювати й редагувати команди;
- подавати конкурсні роботи через GitHub-репозиторій;
- спілкуватися всередині платформи;
- керувати профілем, статусом і медіаоформленням сторінки;
- отримувати сповіщення про події та дії в системі.


Окремі ролі платформи отримують розширені можливості:


- **Учасник** — робота з командами, профілем, чатом і поданням робіт;
- **Журі** — перегляд турнірів і оцінювання submissions;
- **Організатор** — керування турнірними процесами;
- **Адміністратор** — модерація користувачів, команд, заявок, чатів, ролей і системних сутностей.


## Ключові можливості


### Публічна частина


- головна сторінка з презентаційними секціями платформи;
- блоки з перевагами, FAQ, відгуками та контактами;
- окрема сторінка відгуків;
- таблиця лідерів;
- обробка `404` і сторінка блокування користувача.


### Авторизація та акаунт


- вхід і реєстрація;
- підтвердження email за кодом;
- OAuth-вхід через зовнішні провайдери;
- збереження токена й сесії на клієнті;
- робота з профілем користувача, аватаром і банером;
- зміна пароля та видалення акаунта.


### Особистий кабінет


- overview-панель з навігацією за основними розділами;
- керування командами й учасниками;
- реєстрація команд на турніри;
- подання та оновлення конкурсної роботи;
- перегляд турнірної інформації та лідербордів;
- вбудовані сповіщення в реальному часі;
- система статусів користувача: `online`, `away`, `offline`.


### Командна та турнірна логіка


- робота з турнірами та їхніми статусами;
- перегляд раундів турніру;
- надсилання GitHub-репозиторію, гілки, demo та pitch video;
- перехід у командний чат;
- перегляд результатів і турнірного прогресу.


### Адміністрування та модерація


- керування користувачами та ролями;
- керування командами;
- обробка заявок організаторів;
- керування чат-кімнатами та їхніми налаштуваннями;
- закріплення повідомлень, анонси й mute-функції;
- керування бейджами та системною статистикою.


## Технологічний стек


- **React 19** — побудова інтерфейсу;
- **Vite** — dev-server і production build;
- **React Router DOM** — маршрутизація;
- **Socket.IO Client** — realtime-взаємодія;
- **ESLint** — контроль якості коду;
- **Vitest + Testing Library** — тестування інтерфейсу;
- **SVGR** — імпорт SVG як React-компонентів.


## Структура інтерфейсу


Проєкт поділено на дві основні зони:


1. **Landing / Public UI**
2. **Dashboard / Private UI**


Основні маршрути:


- `/` — головна сторінка платформи;
- `/login` — вхід;
- `/register` — реєстрація;
- `/verify-email` — підтвердження email;
- `/dashboard` — особистий кабінет;
- `/dashboard/profile/:username` — публічний профіль користувача;
- `/leaderboard` — таблиця лідерів;
- `/reviews` — відгуки;
- `/banned` — сторінка обмеженого доступу;
- `*` — `404 Not Found`.


## Архітектура проєкту



## Інтеграція з backend


Frontend працює з backend API та підтримує два режими:


- **через Vite proxy** для локальної розробки;
- **через прямий backend URL** для OAuth-redirect і окремих API-викликів.


У проєкті передбачений dev-режим без backend, який дозволяє використовувати mock-дані для частини користувацьких сценаріїв.


## Змінні середовища


Приклад конфігурації:


```env
VITE_APP_NAME=Code League
VITE_APP_VERSION=0.0.0
VITE_FRONTEND_PORT=3000
VITE_API_BASE_URL=/api
VITE_BACKEND_URL=http://localhost:3001
```


Додатково в проєкті використовуються dev-прапори:


- `VITE_CHECK_BACKEND=false` — запуск частини інтерфейсу без backend;
- `VITE_DEBUG_API=true` — детальний debug API-запитів у консолі.


## Встановлення та запуск


### 1. Встановлення залежностей


```bash
npm install
```


### 2. Запуск у режимі розробки


```bash
npm run dev
```


### 3. Production build


```bash
npm run build
```


### 4. Локальний preview production-версії


```bash
npm run preview
```


### 5. Перевірка якості коду


```bash
npm run lint
```


## NPM-скрипти


| Скрипт | Призначення |
|---|---|
| `npm run dev` | запуск dev-сервера Vite |
| `npm run build` | production-збірка |
| `npm run preview` | локальний перегляд build-версії |
| `npm run lint` | перевірка ESLint |
